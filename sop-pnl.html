🎯 1. Overview & Purpose

The Profit & Loss OS provides a real-time financial performance view for the business, built directly from reconciled transactions stored in Airtable. It renders a multi-section P&L grid, nine KPI cards, four interactive charts, and an AI-powered analysis panel — all from the same underlying dataset.

ℹ️
Scope: Covers Revenue, Cost of Goods Sold (COGS), and Operating Expenses (OpEx) for the Real Estate and Operations Director businesses. Capital expenditure, loans, transfers, balance-sheet items, and personal transactions are explicitly excluded — they carry sub-categories not in the allow-list and are silently skipped.
⚠️
Dependency: The P&L page reads from the global allTransactions array, which is loaded by the Leadership Dashboard tab. If you navigate directly to the P&L tab without first opening the Dashboard, the page will display a loading message. Open the Dashboard tab first to populate the data, then switch back.

The primary financial targets the report measures against:

MetricMonthly TargetNotes
Revenue £35,000/mo Constant PNL_REVENUE_TARGET = 35000
Gross Profit Margin 80% Constant PNL_GROSS_MARGIN_TARGET = 80
Net Profit £5,000/mo Constant PNL_CLEAR_PROFIT_TARGET = 5000
Net Profit Margin 15% Constant PNL_NET_MARGIN_TARGET = 15
Maintenance Budget £3,000/mo Constant PNL_MAINT_TARGET (falls back to MAINT_TARGET_GBP = 3000 from config.js)
Wages Budget £1,500/mo Constant PNL_WAGES_TARGET (falls back to WAGES_TARGET_GBP = 1500 from config.js)

🔑 2. How to Access

1

First-time setup

one-off
  1. Open index.html in your browser.
  2. On the authentication screen, paste your Airtable Personal Access Token (PAT) into the input field and press Enter or click the sign-in button.
  3. The token is saved to localStorage under the key _dlr_pat and will be used automatically on future visits.
  4. The Leadership Dashboard loads, populating allTransactions and other global arrays.
2

Navigate to P&L

every visit
  1. In the left sidebar, click 💰 Profit & Loss OS. This calls switchTab('pnl').
  2. The tab panel id="tab-pnl" becomes visible and renderPnL() is called.
  3. The report builds immediately from the in-memory allTransactions array — no additional network request is made.
  4. Charts render on the next animation frame via requestAnimationFrame to avoid layout thrash.
💡
You can also deep-link directly to index.html#pnl. The platform will restore the PAT from localStorage and switch to the P&L tab automatically.

📋 P&L Grid

Below the charts sits a scrollable spreadsheet-style table showing each sub-category by month column. The header row (month labels) and the leftmost column (line item names) are both sticky — they freeze in place when you scroll horizontally or vertically.

  • The grid is constrained to max-height: 85vh with internal scroll.
  • Section headers (Revenue, Cost of Goods Sold, Operating Expenses) appear as dark full-width rows in uppercase.
  • Total rows (Total Revenue, Total COGS, Gross Profit, Total Operating Expenses, Net Profit) appear in colour-coded rows.
  • Margin rows (Gross Profit Margin %, Net Profit Margin %) appear in italic below their corresponding profit rows.
  • Any cell that has underlying transactions is clickable and opens the drill-down modal (see Section 7).
  • A summary footnote below the grid restates the active targets for reference.

🤖 6. AI Analysis Feature

An AI analysis panel sits at the top of the page, above the KPI cards. It is triggered manually and calls the Claude proxy with a structured JSON summary of the current P&L data.

1

Generating an Analysis

manual trigger
  1. Click Generate Analysis in the AI panel. The button calls pnlRunAIAnalysis().
  2. The panel immediately shows three animated dots labelled Analysing P&L data… while the request is in flight.
  3. The function builds a JSON context object (pnlBuildAIContext()) containing: business name, period, all six targets, grand totals, a month-by-month breakdown (revenue / COGS / OpEx / gross profit / gross margin / net profit / net margin per month), and the top 10 expense sub-categories by period total.
  4. The request is sent via POST to the AI proxy endpoint (AI_PROXY, defined in js/ai-assistant.js as https://claude-proxy.kevinbrittain.workers.dev). Model: claude-sonnet-4-20250514, max tokens: 1,500.
  5. The response is rendered using renderMarkdown() (defined in js/ai-assistant.js) so bold text, bullet lists, and headers display correctly.
  6. A ↻ Refresh Analysis button appears below the output to regenerate at any time.
📝 AI Response Structure

The system prompt instructs the AI to return its analysis in exactly this structure, in British English, under 350 words:

  1. Performance Summary — 2–3 sentences on overall position
  2. Key Trends — 3–4 bullet points on patterns across the period
  3. Areas of Concern — 2–3 bullets on problems or risks
  4. Opportunities — 2–3 bullets on cost savings or revenue improvements
  5. Path to 15% Net Margin — specific actions with £ amounts needed to reach the net margin target
💡
The AI is instructed to compare against all six targets (revenue, gross margin, net profit, net margin, maintenance, wages) and flag any months where net profit was negative or margin dropped sharply.
⚠️
One request at a time: the flag _pnlAiLoading prevents concurrent requests. Clicking Generate Analysis while one is in flight is ignored. If the request fails, a Retry button appears in the panel.

🔍 7. Drill-Down & Editing

Clicking any non-zero cell in the P&L grid opens a full-screen modal overlay showing the underlying transactions. The function pnlDrill(scope, section, subCat, monthKey) handles four click types:

Click targetScope valueTransactions shown
A single month cell on a sub-category row cell That sub-category in that specific month only
The Total column on a sub-category row subTotal All months for that sub-category
A month cell on a section total row (e.g. Total Revenue) monthTotal All sub-categories in that section for that month
The Total column on a section total row sectionTotal All sub-categories in that section across all months
ℹ️
Transactions are sorted newest-first by date. The modal header shows the transaction count and the running sum of all Report Amount values.
2

Modal Columns & Inline Editing

ColumnEditable?Notes

Transaction Detail Read-only Vendor name if present; falls back to description (txLabel())

⚠️
Changes save immediately. Each edit fires a PATCH request to Airtable (https://api.airtable.com/v0/{BASE_ID}/tbln0gzhCAorFc3zB/{txId}) using returnFieldsByFieldId: true. A ✓ appears in the status column on success, or ✗ error message on failure. There is no undo.
💡
Auto-refresh on close: If any edit was saved during the modal session (tracked by _pnlEditsMade), closing the modal automatically calls renderPnL() to reflect the changes in the grid. The modal closes when you click outside it, press Escape, or click the × button.

📅 8. Filtering & Date Range

Two dropdown selectors appear at the top-right of the page heading. Both call renderPnL() immediately on change — no submit button is needed.

SelectorID in DOMOptionsDefaultState variable
Business pnlBizSelect Real Estate · Operations Director Real Estate pnlBusinessName
Period pnlMonthsSelect 1 month · 3 months · 6 months · 12 months 12 months pnlMonths
ℹ️
How trailing months work: The function pnlMonthKeys(n) generates n month keys ending with the current calendar month. For example, with Period = 3 and today being April 2026, it returns Feb 2026, Mar 2026, Apr 2026. Months are expressed as YYYY-MM keys internally and displayed as MMM YY in headers (e.g. Apr 26).
ℹ️
Business filtering: The buildPnL() function resolves each transaction's Business (For Reports) linked field and includes only transactions where the resolved name matches the selected business exactly. The available businesses are hard-coded to ['Real Estate', 'Operations Director'] in the constant PNL_TOP_BUSINESSES.

PATCH API call (inline edit)

When an edit input changes in the drill-down modal, pnlEditTxField() makes the following call:

  • Method: PATCH
  • URL: https://api.airtable.com/v0/appnqjDpqDniH3IRl/tbln0gzhCAorFc3zB/{recordId}
  • Auth: Authorization: Bearer {PAT}
  • Body: { "fields": { "{fieldId}": ["{linkedRecordId}"] }, "returnFieldsByFieldId": true }
  • Clear field: To unlink, the body sends an empty array: { "{fieldId}": [] }
  • On success: The in-memory allTransactions record is updated with the returned fields. Status indicator shows ✓ in green.
  • On error: Status indicator shows ✗ error message in red. The Airtable error message is extracted from the response JSON where possible.

🛠️ 11. Troubleshooting

❌ P&L shows “Loading transactions… if this persists, open the Leadership Dashboard first”
✅ The global allTransactions array is empty. Switch to the Leadership Dashboard tab and wait for data to load (look for KPI cards to populate). Then switch back to P&L. The data is loaded once per session and cached in memory.
❌ A transaction appears in the wrong P&L section or is missing entirely
✅ The transaction's Sub-Category or Business link is incorrect or missing in Airtable. Click the value in the relevant area of the grid to open the drill-down modal, then use the type-to-search inputs to correct the Sub-Category and/or Business. Changes save to Airtable immediately. The grid refreshes automatically when you close the modal.
❌ Revenue or an expense line shows zero when you expect a value
✅ Check that the transaction in Airtable has both a Sub-Category (must match one of the 27 in the allow-list) and a Business (For Reports) link pointing to the business currently selected in the dropdown. Also confirm the transaction's Date falls within the selected period. Any of these being absent causes the transaction to be silently excluded.
❌ Drill-down edit shows “✗ no match” after typing
✅ The typed name did not match any record in the lookup table (case-insensitive comparison). Check spelling carefully — the name must exactly match an existing Category, Sub-Category, or Business record in Airtable. Use the browser's autocomplete suggestions from the datalist for guidance.
❌ Drill-down edit saves but the P&L grid does not update after closing
✅ This should not happen — _pnlEditsMade is set to true on any successful save, and pnlCloseDrill() calls renderPnL() automatically. If it does occur, change the Period dropdown to a different value and back to force a full re-render.
❌ AI Analysis fails with “Analysis unavailable: API 5xx”
✅ The Claude AI proxy at https://claude-proxy.kevinbrittain.workers.dev is temporarily unavailable. Click the Retry button. If the error persists, check your internet connection and try again in a few minutes.
❌ Charts are blank or not rendering
✅ Chart.js failed to load from CDN. Hard-refresh the page (Cmd+Shift+R on Mac / Ctrl+Shift+R on Windows) to force the browser to re-fetch all scripts. If you are offline, charts will not render — all other P&L data is unaffected.
❌ KPI card shows wrong target indicator (e.g. green when it should be red)
✅ Cost cards (COGS, OpEx, Maintenance, Wages) use invertComparison: true, meaning lower = green. Verify you are reading the sub-label which shows the budget amount for context. If the targets themselves seem wrong, check the constants at the top of js/pnl.js: PNL_REVENUE_TARGET, PNL_GROSS_MARGIN_TARGET, PNL_NET_MARGIN_TARGET, PNL_CLEAR_PROFIT_TARGET.
