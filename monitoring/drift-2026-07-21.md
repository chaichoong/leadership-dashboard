### 4g FAIL — CRITICAL: duplicate `CAT_NAME_FIELD` stops the app loading

14 of 61 sync-invariant tests fail, all with the same root cause. Every one reports the same
uncaught page error:

```
Identifier 'CAT_NAME_FIELD' has already been declared
```

The constant is declared twice at top level, and because every `js/*.js` file shares one global
scope (plain `<script>` tags, no bundler), a repeated `const` is a **SyntaxError that kills the
whole page**, not just one tab:

- [js/config.js:407](js/config.js:407) — `const CAT_NAME_FIELD = 'fldii4oUzSfmplihO';` (new,
  **uncommitted**)
- [js/costs.js:98](js/costs.js:98) — `const CAT_NAME_FIELD = 'fldii4oUzSfmplihO';` (existing,
  committed)

Same field ID, so this is a duplicated definition, not a conflict of meaning.

Failing tests: `no-crash-on-load` (overview, cfv, invoices, costs, fintable, sitemap, pnl,
transactions, plus tab-switching), `auth-token-handling`, `overdue-early-payment` ×3,
`skills-settings-request`.

**Blast radius if it ships: total.** Every tab, not one feature.

**Production is safe right now — verified three ways:**
- `git show HEAD:js/config.js | grep -c CAT_NAME_FIELD` → `0`
- live `js/config.js` on GitHub Pages → `0` occurrences
- live `index.html`, `js/config.js`, `js/wealth.js` all return HTTP 200

**Not fixed by this run, deliberately.** `js/config.js` is a shared file currently open in
