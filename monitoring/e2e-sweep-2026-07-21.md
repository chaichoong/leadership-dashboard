## ⚠️ Caught on the way out: an app-killing bug sitting UNCOMMITTED in the working tree

Production is not affected. This is not live. But it is one `git push` away from taking the
whole app down, so it is recorded here.

Pushing this report tripped the pre-push gate: **14 sync-invariant tests failed**, all with the
same fatal error:

    SyntaxError: Identifier 'CAT_NAME_FIELD' has already been declared

**Cause.** The uncommitted edit to `js/config.js` (another session's in-flight work, added to
support `js/wealth.js:1150`) declares:

    js/config.js:407   const CAT_NAME_FIELD = 'fldii4oUzSfmplihO';

but that constant already exists at:

    js/costs.js:98     const CAT_NAME_FIELD = 'fldii4oUzSfmplihO';

Both load as plain `<script>` tags into one shared scope, so the second `const` is a **fatal parse
error**. It does not break one tab, it stops the app booting at all — every tab, dead.

**Proof it is local-only, not live:**
- Production `js/config.js` contains 0 occurrences of `CAT_NAME_FIELD`; only `js/costs.js` declares it.
- This morning's prod walk across all 25 tabs recorded **0 page errors**.
- Tests run against **committed `main`** in a clean worktree: **61/61 passed**. Run against the dirty
  working tree: 14 failed. The delta is entirely the uncommitted change.

**The fix is one line** — delete the new declaration in `js/config.js` and let `wealth.js` use the
existing one from `costs.js`, or rename it if the two are meant to be independent. Not applied here:
`js/config.js` and `js/wealth.js` belong to another live session, and this sweep is read-only.

**The real risk is the bypass.** The gate is doing its job. If that session reads the red as noise
and pushes with `SKIP_SYNC_TESTS=1`, the app goes down platform-wide on the next deploy.

## Warnings (non-blocking, no Slack sent)

1. **Cold boot takes ~40 seconds.** From navigation to the loading overlay clearing was 40.1s on a
   cold cache (8,744 transactions fetched up front). No error, and it completes reliably, but it is
   slow enough that a first-time user could think the app has hung. Worth a look if it drifts higher.
2. **`ai-brain` iframe: 2 × `ERR_UNKNOWN_URL_SCHEME`.** The tab renders fine (3,008 chars). No
   custom scheme exists in `ai-brain.html` source, so this is almost certainly a link pointer coming
   from the private brain feed data (e.g. an `obsidian://`-style URL) that the headless browser
   cannot resolve. Cosmetic in a real browser where the scheme is registered.
3. **`sitemap`: 11 pages have an SOP version behind `pageVer`** (overview, os-strategy, tasks, …).
   Documentation drift, not app breakage — SOPs have not been re-bumped to match auto-bumped page
   versions. The second warn ("GitHub API reachable — not yet fetched") is just lazy-load timing.

## Sweep tuning notes

- The registry now returns **25** tabs (was 24) — `prospecting` was added and was covered
  automatically by live enumeration, as intended.
- The sweep prompt's boot check (`window.PAT`, `window.allTransactions`) is wrong: those globals are
  IIFE-scoped, not on `window`. A DOM-based wait (loading overlay hidden **and** `allTransactions.length > 0`,
  polled up to 180s) is the correct signal — 30s is not enough for a cold boot.
- The prompt assumes every `PAGE_REGISTRY` id has a `tab-<id>` panel. Two entries
  (`compliance`, `how-it-works`) are `standalone:` pages instead; they must be checked by URL, not panel.
- The PAT was read from disk inside a Playwright script rather than injected through a browser
  tool argument, so the token never entered the session transcript. Recommend keeping that method.
