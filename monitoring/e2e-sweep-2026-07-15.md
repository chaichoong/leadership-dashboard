## Failures needing attention

1. **CFV tab hangs (FAIL).** Switching to Cash Flow Voids leaves it stuck on "LOADING CASH FLOW VOIDS…" and never renders. `renderCFVTab()` freezes the main thread for 30s+. A code comment in `detectCFVs` ("was the main freeze") shows a prior freeze was partly addressed — this looks like a regression or an incomplete fix. Rent-arrears / void tracking is unusable from this tab until fixed. Note: CFV summary counts still surface on the overview/income tabs, so the underlying detection works; the dedicated tab's full render is what freezes.

2. **compliance & how-it-works blank the app on the Site Map "Open" link (WARN).** Both are `standalone` pages that load correctly via their direct `.html` links, but `switchTab('<id>')` has no panel to show and clears the screen. Fix options: point the Site Map "Open" links at the standalone `.html` (open in-place/new tab) instead of `switchTab`, or give `switchTab` a fallback that opens the standalone file when no `tab-<id>` panel exists.
