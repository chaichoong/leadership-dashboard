📋 1. Overview

The Property Compliance system monitors mandatory safety certifications across the full property portfolio. It has two integrated components:

  • Compliance Dashboard (compliance.html) — a read-only matrix showing current cert status for every property, with colour-coded expiry warnings.
  • Gmail Integration (via the Inbound Comms Tracker, follow-up.html) — when compliance emails arrive in Gmail, labelling them as 10: property compliance triggers AI extraction of certificate details, which are automatically written to Airtable.

Together they create a hands-off pipeline: emails arrive, get labelled, AI extracts the details, Airtable is updated, and the dashboard reflects the latest status.

🔗 2. How to Access

Compliance Dashboard
  1. Open the Leadership Dashboard in your browser.
  2. Click the Compliance tab in the top navigation bar.
  3. If prompted, enter your Airtable Personal Access Token (PAT). This is saved in your browser and shared with the Inbound Comms Tracker automatically.
  4. The compliance matrix loads, showing all properties with their certification statuses colour-coded.
  5. Click Refresh in the top-right corner to reload data at any time.
Inbound Comms Tracker (Email Processing)
  1. Open the Leadership Dashboard and click the Inbound Comms tab.
  2. Connect your Gmail account and configure your Airtable PAT in Settings (if not already done).
  3. Select the 10: property compliance label from the label dropdown to view compliance emails.
💡 The Airtable PAT is shared between the Compliance Dashboard and Inbound Comms Tracker. Enter it once in either page and it works in both.

📜 3. Certificate Types

The system tracks three main certificate types, displayed as columns on the dashboard:

Gas Safety Certificate (GSC)

Renewal: Annual (every 12 months)

Required for all properties with gas appliances. Blocks of apartments: N/A at unit level if the building has no individual gas supply (shown as N/A on dashboard).

Electrical Installation (EICR)

Renewal: Every 5 years

Mandatory for all rental properties. In blocks, each apartment has its own EICR.

Landlord Insurance

Renewal: Annual

Building-level cover. In blocks, shown at the property level rather than per-apartment.

Fire Alarm

Tracked in Airtable but does not have a dedicated dashboard column. Emails mentioning fire alarm certificates are still processed and stored.

🔨 4. Dashboard Features

What You See
  • Summary Bar — Four KPI cards at the top: Expired (red), Expiring within 30 days (amber), Active (green), and No Cert (grey).
  • Self-Managed Properties — The main table showing each property as a row with three cert columns (GSC, EICR, Insurance). Each cell is colour-coded and shows the renewal date.
  • Agent-Managed Properties — A separate section below, grouped by managing agent (e.g. Roc Immo, Intus, Cheffins). These are shown for visibility but managed by the agent.
  • Blocks & Apartments — Properties marked as “Block” in Airtable expand into their individual units, each with its own cert row.
  • Airtable Links — Click any cert cell to open that record directly in Airtable for editing.
  • Refresh Button — Top-right corner, reloads all data from Airtable without a full page refresh.
  • Back to Dashboard — Returns to the main Leadership Dashboard overview tab.
ℹ️ The dashboard always shows the latest (most recent renewal date) certificate per property and type. Older certs are preserved in Airtable as an audit trail but do not appear on the dashboard.

Status Colour Codes

Active — valid, more than 30 days remaining
Expiring — 30 days or fewer remaining
Expired — renewal date has passed

📧 5. Gmail Integration

How It Works

The Inbound Comms Tracker processes compliance emails automatically when they are labelled in Gmail.

📧 Email arrives Gas cert, EICR, insurance
⟶
🏷 Label applied 10: property compliance
⟶
🤖 AI extraction Claude Haiku via proxy
⟶
✅ Airtable updated New cert created

Step-by-Step

  1. A compliance email arrives in Gmail (e.g. gas safety certificate from an engineer, EICR report, insurance renewal confirmation).
  2. In the Inbound Comms Tracker, use the label dropdown or AI suggestion to apply the 10: property compliance label. This triggers processing automatically.
  3. The system sends the email subject and body to Claude Haiku, which extracts: cert_type (GSC, EICR, Landlord Insurance, or Fire Alarm), property_name, renewal_date, and a brief summary.
  4. The extracted data is matched against your Airtable Properties table. If a match is found, a new cert record is created in the Certs table with a link back to the original Gmail message.
  5. Any older certs of the same type for the same property are automatically marked as “Expired” (see Audit Trail below).
  6. The compliance dashboard immediately reflects the new cert on the next Refresh.

Bulk Processing

When viewing the 10: property compliance label, a compliance banner appears showing how many emails have been processed vs pending. Click Process All Pending to batch-process all unprocessed emails in one go.

⚠️ The AI can only extract details that are present in the email text. If an email just says “certificate attached” with no property name or date, it will show as “No Match” and need manual entry in Airtable.

📚 6. Audit Trail

The system maintains a full audit trail of every certificate ever received. Old records are never deleted.

How It Works

  • When a new compliance email is processed, a new cert record is always created in Airtable (never overwritten).
  • After creation, any older certs of the same property and type are automatically set to “Expired” status.
  • The compliance dashboard only shows certs with the latest renewal date per property and type — expired records are filtered out.
  • All historical records remain in Airtable, searchable and traceable, each with a link to the original Gmail message.
💡 To view the full certificate history for a property, open the property record in Airtable and look at all linked cert records. Each one includes the Gmail link and processing date.

🏢 7. Blocks & Apartments

Properties with a “Block” type in Airtable are handled differently from standard houses or flats:

Cert Type Block-Level or Per-Unit? Notes
GSC N/A for most blocks Unless individual apartments have their own gas supply, GSC shows as N/A for block units
EICR Per unit Each apartment within a block has its own EICR record and expiry date
Insurance Building-level One policy covers the whole block. Applied at the property level and inherited by all units
ℹ️ On the dashboard, blocks expand into rows for each apartment. Units are listed alphabetically beneath the block name.

👥 8. Agent-Managed Properties

Properties with a managing agent recorded in Airtable appear in a separate “Agent-Managed” section at the bottom of the dashboard. They are grouped by agent name and sorted alphabetically.

Agent-managed properties are excluded from the summary KPI counts at the top of the dashboard, since compliance responsibility sits with the agent. However, they are still shown so you can monitor overall portfolio status.

Adding or Updating Agent Info

To mark a property as agent-managed, set the Agent field in the Airtable Properties table. The dashboard recognises the following agents and automatically groups them:

🗃 9. Data Source & Airtable

Airtable Structure

The system reads from three tables in the Operations Director base:

Table Purpose Key Fields
Properties All properties in the portfolio Property Name, Short Name, Property Type (Standard/Block), Agent, linked Units
Units Individual apartments within blocks Unit Name, linked Property
Certs Certificate records (one per cert received) Type (select: GSC, EICR, Landlord Insurance, Fire Alarm), linked Property, linked Unit, Status (Active/Expired), Renewal Date, Notes (contains Gmail link)

Adding a Certificate Manually

  1. Open the Certs table in Airtable.
  2. Create a new record.
  3. Set the Type (GSC, EICR, or Landlord Insurance).
  4. Link it to the correct Property (and Unit if applicable for blocks).
  5. Set the Renewal Date (the date the cert expires or is next due).
  6. Set Status to Active. The dashboard will pick it up on next refresh.
ℹ️ You only need to add certs manually for situations not covered by email (e.g. certificates received by post). Email-based certs are created automatically via the Gmail integration.

🔧 10. Troubleshooting

Dashboard shows “No Cert” despite having a valid certificate
Check the Certs table in Airtable. Ensure the cert record has the correct Property linked, the correct Type selected, and a Renewal Date set. Also check that Status is “Active” (not “Expired”). Click Refresh on the dashboard.
Email processed but shows “No Match”
The AI could not match the property name in the email to any property in Airtable. Check that the property name or short name in Airtable matches what appears in the email. You may need to add a short name alias.
Email processing failed
This usually means the AI could not extract valid JSON from the email content, or the Airtable API returned an error. Try the “Process All Pending” button to retry. If it persists, check the browser console for detailed error messages.
PAT authentication prompt keeps appearing
Your Airtable Personal Access Token may have expired or been revoked. Generate a new one from airtable.com/create/tokens with read/write access to the Operations Director base, then enter it in Settings.
Property missing from the dashboard entirely
The property must exist in the Airtable Properties table. Add it there first, then create cert records linked to it. Properties appear on the dashboard regardless of whether they have certs (they just show blank cells).
Old expired certificate still showing on the dashboard
The dashboard filters out certs with “Expired” status and shows only the latest renewal date. If an old cert is still showing, check that its Status in Airtable is correctly set to “Expired”, or that a newer Active cert exists.
Compliance label not appearing in the Inbound Comms Tracker dropdown
Ensure the Gmail label is named exactly 10: property compliance (with the number prefix). The dropdown shows labels matching the pattern N: label name where N is a number. Refresh the page after creating the label in Gmail.
