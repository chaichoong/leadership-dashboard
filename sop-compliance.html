📋 1. Overview & Purpose

The Property Compliance OS (compliance.html) is a read-only dashboard that displays the current status of mandatory safety certificates across your entire rental portfolio. It connects directly to Airtable and renders live data on load.

The dashboard answers one question at a glance: which properties have expired, expiring, or missing certificates?

  • Three certificate types are tracked per property: Gas Safety (GSC), EICR, and Landlord Insurance.
  • Properties are split into self-managed (your action list) and agent-managed (monitoring only).
  • Block properties expand into per-unit rows with individual EICR tracking.
⚠️
Legal obligation: Operating a rental property without a valid gas safety certificate is a criminal offence. The dashboard flags expired GSCs in red with highest priority.
ℹ️
Read-only: This dashboard only reads from Airtable. To update certificate records, edit them directly in Airtable or use the manual entry process (section 9). There is no write functionality in compliance.html.

🔑 2. How to Access

🚪 Opening the Dashboard
  1. Open compliance.html directly, or navigate via the Leadership Dashboard sidebar → Compliance.
  2. If not already authenticated, an auth screen appears. Enter your Airtable Personal Access Token (PAT) and click Connect.
  3. The PAT is stored in two places: sessionStorage key _dlr_pat (for this tab) and localStorage key airtable_pat (shared with the Inbound Comms Tracker). This means if you have already authenticated in the Inbound Comms Tracker, Compliance will pick up the same token automatically.
  4. Data loads immediately — a full-screen spinner is shown while fetching.
  5. To refresh data at any time, click Refresh in the top-right header.
  6. To return to the main Leadership Dashboard, click Back to Dashboard in the top-right header (calls goToDashboard(), which uses window.parent.switchTab('overview') if loaded in an iframe, or redirects to index.html).
💾
Shared authentication: The PAT is shared between compliance.html and follow-up.html (Inbound Comms) via localStorage key airtable_pat. Enter it once in either page and both pages will use it.

📜 3. Certificate Types

The dashboard recognises three certificate types. These must match exactly in the Type field of the Airtable Certs table:

Type (exact value in Airtable)Typical RenewalNotes
GSC (Gas Safety Certificate) Annual Required for all properties with gas appliances. Block apartments without individual gas supply show N/A on the dashboard.
EICR (Electrical Installation Condition Report) Every 5 years Required for all rental properties. Block apartments each have their own EICR record.
Landlord Insurance Annual Building-level cover. For blocks, insurance is recorded at the property level and applied across all units (units show See building).
ℹ️
Any cert record with a Type value other than GSC, EICR, or Landlord Insurance is silently ignored by the dashboard. Other types (e.g. Fire Alarm) may exist in Airtable but do not appear here.

📊 4. Dashboard Features

📊 Summary Bar

Four KPI cards shown at the top of the dashboard. These counts include only self-managed properties — agent-managed properties are excluded from the totals.

Expired — cert has expired (renewal date in the past)
Expiring (30d) — expires within 30 days
Active — valid, more than 30 days remaining
No Cert — no cert record found for this property/type

Each property’s GSC, EICR, and Insurance cert contributes one count to the summary bar. For block properties, only the EICR count is included (per unit).

📋 Self-Managed Properties Table

The main table lists every self-managed property as a row with three cert columns. Standard (non-block) properties show one row per property. Block properties show a section header row followed by one row per apartment unit.

  • Each cert cell shows the status tag and renewal date (e.g. Active   12 Mar 2027).
  • Clicking any cert cell opens the Airtable record directly in a new tab (link to https://airtable.com/[baseId]/[certsTableId]/[recId]). Only cells with a known recId are linked.
  • Dates within 30 days show in amber; expired dates show in red with a “(Xd overdue)” note.
ℹ️
Only current certs are shown. The dashboard skips any cert record whose Status field in Airtable equals “Expired” (if (c.s === 'Expired') continue). Expired records remain in Airtable for audit purposes but are not rendered. If multiple active certs exist for the same property and type, the one with the latest renewal date is shown.
🤝 Agent-Managed Section

Agent-managed properties appear in a collapsible section below the main table (using an HTML <details> element). Click the summary row to expand or collapse it.

Within the section, properties are grouped by agent name with a purple header row for each agent, showing the count of properties. Within each agent group, properties follow the same standard/block rendering as self-managed.

🎨 5. Status Tags & Row Colours

The cell() function in compliance.html renders each cert cell with one of four tags, determined by the cert’s renewalDate and status fields:

TagConditionDate Colour
Expired status === 'Expired' OR renewal date is in the past (days < 0) Red, bold. Shows “(Xd overdue)” in red below the date.
Expiring Renewal date is within 30 days (days ≥ 0 and days ≤ 30) Amber, bold. Shows “(Xd left)” in amber below the date.
Active Renewal date is more than 30 days in the future Normal (dark grey), no extra label
— No cert record found for this property and type N/A (grey dash)
N/A Not applicable (e.g. GSC for block apartments, Insurance for block unit rows) N/A (transparent/grey)
See building Insurance for block apartment units (covered at property level) Shown as “See building”
No date Cert record exists with Active status but no renewal date set Grey, no date displayed
🎨 Row Background Colours

Entire table rows are highlighted based on the rowClass() function:

ClassTriggerBackground
row-expired Any of the three cert cells is Expired (status = Expired or days < 0) Light red background (#fff5f5), red hover (#fff0f0)
row-expiring No expired certs, but any cert expires within 30 days Light amber background (#fffdf5), amber hover (#fffbeb)
(no class) All certs are Active or not applicable Standard white background
ℹ️
Row colours are evaluated left-to-right: row-expired takes priority over row-expiring. Only the first matching condition is applied.

🏗 6. Blocks & Apartments

Properties with propType = 'Block' in the Airtable Properties table are handled differently from standard properties.

🏢Block PropertypropType = Block
➡️

➡️

EICR Property-level cert record Unit-level cert record (linked to the specific unit record in the Units table)
Landlord Insurance Property-level cert record See building — insurance covers the whole block
💡
Cert propagation: If an EICR record is linked to the block property rather than a specific unit, the dashboard propagates it to all units of that block. This means a single property-level EICR can pre-fill all unit rows until individual unit-level certs are added.

🤝 7. Agent-Managed Properties

A property is considered agent-managed if the Agent field (FP.agent) in the Airtable Properties table contains one of the following strings (case-insensitive):

  • roc immo
  • intus
  • mears
  • cheffins
  • sequence
  • simon collins

This check is performed by the isAgentManaged(agentStr) function in compliance.html.

Agent-managed properties are:

  • Excluded from the summary bar KPI counts (self-managed only).
  • Shown separately in the collapsible “Agent-managed” section, grouped by agent name with an indigo group header.
  • Rendered using the same table structure as self-managed properties.
ℹ️
To add a new recognised agent, update the AGENT_NAMES array in compliance.html and the corresponding Airtable Property record.

✏️ 9. Manual Certificate Entry

Certificates received by post or any channel other than email must be entered manually in Airtable. The dashboard will pick them up on the next Refresh.

  1. Open the Certs table in Airtable (base: Operations Director).
  2. Create a new record.
  3. Set Type to exactly one of: GSC, EICR, or Landlord Insurance (must match exactly).
  4. Link to the correct Property. For block apartments, also link to the specific Unit.
  5. Set Renewal Date to the certificate’s expiry or next-due date.
  6. Set Status to Active.
  7. Return to the compliance dashboard and click Refresh.
⚠️
Do NOT set the Status to “Expired” for a newly entered certificate — that will cause the dashboard to skip it entirely. Only set Status to “Expired” when replacing a certificate with a newer one, to preserve the audit trail.

🛠️ 10. Troubleshooting

Dashboard shows auth screen even after entering PAT
Ensure the PAT has read access to the Operations Director Airtable base and covers the Certs, Properties, and Rental Units tables. Generate a new one at airtable.com/create/tokens if needed. The error message “Invalid token” appears if the API returns 401 or 403.
Property shows all blank cells (no cert data)
The property exists in Airtable but has no cert records linked to it, or all its cert records have Status = “Expired”. Add cert records manually (section 9) or check that existing records link to the correct property ID.
Certificate shows as “Expired” despite being valid
The cert record in Airtable may have Status = “Expired”. The dashboard skips these records. Check the Certs table in Airtable and set Status to “Active” if the certificate is still valid.
A property is missing from the dashboard entirely
The property must exist in the Airtable Properties table. Add it there, then add cert records linked to it. The dashboard shows all properties regardless of whether they have certs.
Block property shows as a single row instead of per-unit rows
Check that the Property Type field in Airtable is set to “Block” (not “Standard”) and that the block property has units linked in its Units field.
Agent-managed property appears in the self-managed section
The Agent field in Airtable must contain one of the recognised agent names: roc immo, intus, mears, cheffins, sequence, or simon collins (case-insensitive partial match). If the agent name uses a different spelling, update it in Airtable to match.
Cert cell is not a clickable link
The cell only shows a link if a cert record was found (recId is known). Cells showing “N/A”, “See building”, or the grey dash have no associated record and therefore no link.
