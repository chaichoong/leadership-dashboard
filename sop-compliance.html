📋 1. Overview

The Property Compliance module tracks all mandatory safety and regulatory certifications across the portfolio. It provides a single view of compliance status per property, highlighting items that are approaching expiry or already overdue.

Compliance items tracked include:

Gas Safety Certificate (CP12)

Annual requirement for all properties with gas appliances. Must be renewed before the expiry date.

Electrical Installation (EICR)

Required every 5 years. Must be satisfactory or have remedial works completed within 28 days of the report.

Energy Performance Certificate (EPC)

Valid for 10 years. Must be rated E or above for rental properties (minimum requirement).

HMO / Selective Licence

Required for Houses in Multiple Occupation or properties in selective licensing areas. Typically valid for 5 years.

⚠️ Operating without a valid gas safety certificate is a criminal offence. The dashboard flags any property where the CP12 has expired in red with highest priority.

🔗 2. How to Access

  1. Open the Leadership Dashboard in your browser.
  2. Click the Compliance tab in the navigation bar.
  3. The compliance matrix loads, showing all properties with their certification statuses colour-coded.

🔨 3. Features

Dashboard Features
  • Compliance Matrix — A grid showing each property as a row and each compliance type as a column. Cells are colour-coded: green (valid), amber (expiring within 30 days), red (expired).
  • Summary KPIs — Header cards showing total compliant properties, items expiring soon, and overdue items.
  • Expiry Timeline — Upcoming expiry dates sorted chronologically so renewals can be planned in advance.
  • Certificate Links — Click a cell to view the uploaded certificate document (PDF stored in Airtable attachments).
  • Filtering — Filter by property, compliance type, or status (Valid / Expiring / Expired).
💡 Use the Expiry Timeline to batch-schedule renewals. Booking multiple gas checks in the same week is more cost-effective than individual visits.

🗃 4. Data Source

Airtable Configuration
  • Base: Operations Director
  • Tables: Properties (for property details) and Compliance (for certificate records)
  • Relationship: Each compliance record is linked to a property via the Property field
  • Key fields: Compliance Type, Issue Date, Expiry Date, Status, Certificate (attachment), Engineer/Provider
ℹ️ When a new certificate is uploaded, update the Issue Date and Expiry Date fields. The dashboard calculates status (Valid/Expiring/Expired) automatically from the Expiry Date.

🔧 5. Troubleshooting

Property showing as non-compliant despite valid certificate
Check the Expiry Date field in the Airtable Compliance record. Ensure the date is correct and in the future. Also verify the compliance type matches the expected category.
Certificate PDF not loading when clicked
Airtable attachment URLs expire periodically. Re-upload the PDF to the Compliance record attachment field. Refresh the dashboard after uploading.
Property missing from the compliance matrix
The property must exist in the Properties table and have at least one linked Compliance record. Add the property to Airtable if it is new, or create the initial compliance records.
Amber warning not appearing for expiring items
The 30-day threshold is calculated from today's date. Check that the Expiry Date is within 30 days. If the dashboard date seems off, verify browser time zone settings and clear cache.
