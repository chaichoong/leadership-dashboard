📈 3. KPI Summary Cards

The dashboard header displays four real-time KPI cards summarising the current invoice state:

Total Outstanding

Sum of all invoices in Pending or Approved status. Indicates cash commitment not yet settled.

Awaiting Approval

Count of invoices still in Pending status requiring manual review before payment.

Overdue

Invoices where Due Date < Today and status is not Paid. Triggers escalation.

Paid This Month

Total value of invoices marked as Paid within the current calendar month.

🤖 4. AI Matching Logic

When an invoice email arrives, the AI engine performs two matching tasks:

  1. Supplier Matching — The sender email and invoice text are compared against the Contacts table. A confidence score (0–100%) is assigned.
  2. Cost Code Assignment — Based on historic invoices from the same supplier and the line-item descriptions, a cost code is suggested.
  3. Threshold Check — If confidence ≥ 85%, the match is auto-applied. Below 85%, the invoice is flagged for manual review.
💡 The AI model improves over time. Each manual correction feeds back into the training data for future matching accuracy.

✅ 5. Approve / Reject Flow

Review Workflow
  1. Open the Invoices tab on the Leadership Dashboard.
  2. Invoices in Pending status appear at the top of the table.
  3. Click an invoice row to expand the detail panel showing the PDF attachment, supplier match, and suggested cost code.
  4. Review the details. If correct, click Approve. If incorrect, click Reject and add a note explaining the reason.
  5. Approved invoices move to the payment queue. Rejected invoices are returned for correction or deletion.
⚠️ Invoices over £500 require a second approval step. The system will prompt for confirmation before finalising.

💵 6. Mark as Paid

  1. Once payment has been made via the bank, return to the Invoices tab.
  2. Locate the approved invoice and click Mark as Paid.
  3. Enter the payment date and reference number if applicable.
  4. The status changes to Paid and the record moves to the archive view.
ℹ️ Paid invoices are not deleted. They remain in Airtable for audit and reporting purposes and contribute to monthly spend analysis.

📚 7. Knowledge Base

Key Concepts
  • Invoice vs. Statement — Only invoices (requests for payment) should enter the system. Statements are informational and should not be processed.
  • Duplicate Detection — The system checks invoice number + supplier + amount to prevent duplicate entries. Flagged duplicates appear with an amber warning badge.
  • Recurring Invoices — Regular monthly invoices from known suppliers can be auto-approved if the amount matches the expected value within a 5% tolerance.
  • Credit Notes — Credit notes appear as negative amounts and automatically offset against outstanding invoices from the same supplier.

⚙ 8. Google Apps Script Setup

How the Ingestion Script Works
  1. A time-driven trigger runs every 15 minutes, scanning the Gmail inbox for emails matching the invoice label.
  2. The script extracts sender, subject, date, and any PDF attachments.
  3. Extracted data is sent to the AI matching endpoint for supplier and cost code identification.
  4. A new record is created in the Airtable Invoices table via the Airtable API with the matched data and attachment.
  5. The processed email is moved to the "Invoices/Processed" Gmail label to prevent reprocessing.
⚠️ If the Apps Script quota is exceeded (e.g. too many API calls), ingestion pauses until the next quota reset. Check the Apps Script dashboard for execution logs.

🔧 9. Troubleshooting

Invoice email not appearing in the system
Check that the email has the correct Gmail label applied. Verify the Apps Script trigger is active in the Apps Script dashboard. Review execution logs for errors.
Wrong supplier match
Manually correct the supplier in Airtable. The correction will be fed back to improve future AI matching. Check if the supplier exists in the Contacts table.
Duplicate invoice detected
Review both records. If genuinely duplicate, reject the newer entry. If the supplier has issued two invoices with the same number, add a distinguishing note and approve both.
KPI cards showing stale data
Refresh the dashboard page. If data remains stale, check the Airtable API connection and verify the data fetch function in the dashboard source code.
PDF attachment not loading
Ensure the attachment URL from Airtable has not expired. Re-upload the PDF manually if needed. Check browser CORS settings if viewing cross-origin.
