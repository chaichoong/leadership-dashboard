📊 1. Overview & Data Flow

The Invoices module (v2.0) manages the full invoice lifecycle with Airtable as the single source of truth. Gmail emails labelled "3. to pay" are synced to the Airtable "Dashboard Invoices" table by an Apps Script running every 15 minutes. The Leadership Dashboard reads from Airtable, and all user edits (amounts, payees, references, due dates) write directly back to Airtable via API — so data persists across browsers and devices. AI matching cross-references invoices against bank transactions for one-click payment reconciliation.

📈 3. KPI Summary Cards

The dashboard header displays four real-time KPI summary cards calculated from the Airtable data:

Unpaid Count

Total number of invoices where Status is not "Paid". Shows how many invoices still need attention.

Known Amount Due

Sum of all unpaid invoices that have an Amount value entered. This is the confirmed cash commitment.

Unknown Amounts

Count of unpaid invoices where the Amount field is still empty. These need the user to click and enter the value.

Overdue

Invoices where Due Date < Today and status is not Paid. Highlights invoices that need immediate action.

🤖 4. AI Matching Logic

The AI matching engine cross-references unpaid invoices against the Airtable Transactions table (bank transactions) to find likely payment matches:

  1. Exact Amount Match — The system looks for bank transactions where the amount exactly matches the invoice amount.
  2. Date Window — Matching transactions must fall within a reasonable date range around the invoice due date.
  3. Keyword Matching — Transaction descriptions are compared against the invoice payee name and subject for additional confirmation.
  4. Suggestion Display — When a match is found, an inline suggestion row appears beneath the invoice with the matched transaction details.
💡 The matching runs automatically when the dashboard loads. It only considers transactions that are not already linked to another invoice.

🤖 5. Inline Match Suggestions

When the AI engine finds a likely transaction match for an invoice, a match suggestion row appears directly beneath the invoice row in the table. This allows one-click approval or rejection without leaving the page.

How Inline Suggestions Work
  1. After AI matching runs, invoices with a suggested transaction display a highlighted sub-row showing the matched bank transaction details (date, amount, description).
  2. Click Approve to accept the match — this triggers the full Mark as Paid workflow (see Section 7), linking the transaction and marking the invoice as paid.
  3. Click Reject to dismiss the suggestion — the rejection is saved in Airtable so the same transaction will not be suggested again for this invoice.
💡 Suggestions only appear for invoices that have an Amount entered. If the Amount field is empty, no matching can be performed — enter the amount first.

✅ 6. Approve / Reject Flow

Review Workflow
  1. Open the Invoices tab on the Leadership Dashboard.
  2. All unpaid invoices are displayed (fetched from Airtable where Status != "Paid").
  3. Fill in any missing details by clicking empty fields — Amount, Payee, Ref, and Due Date are all editable inline. Each edit saves directly to Airtable.
  4. If a transaction match suggestion appears beneath an invoice, review the details. Click Approve to accept (triggers mark-paid) or Reject to dismiss (saved in Airtable).
  5. For invoices without an automatic match, use the Mark as Paid button manually once payment has been made.
ℹ️ All edits persist in Airtable. You can close your browser, switch devices, or clear your cache — your data will still be there next time you open the dashboard.

💵 7. Mark as Paid

Marking an invoice as paid triggers three actions simultaneously:

  1. Airtable Update — The invoice record's Status is set to "Paid" and the Paid Date is set to today. If a matched transaction was approved, the Matched Transaction linked record is also set.
  2. Gmail Label Move — The original email is moved from the "3. to pay" Gmail label to "4: paid" via the Apps Script proxy. This keeps Gmail in sync with the dashboard.
  3. Transaction Link — If the payment was matched to a bank transaction (via AI matching or manual selection), the Airtable Transactions record is linked to the invoice.
How to Mark as Paid
  1. Click the Mark as Paid button on the invoice row, OR click Approve on a transaction match suggestion (which triggers mark-paid automatically).
  2. The invoice row disappears from the unpaid list and the KPI cards update immediately.
ℹ️ Paid invoices are not deleted. They remain in Airtable for audit and reporting purposes. The dashboard simply filters them out by only showing records where Status != "Paid".

📚 8. Knowledge Base

Key Concepts
  • Airtable is the source of truth — All invoice data lives in the "Dashboard Invoices" table. The dashboard reads from and writes to Airtable. Nothing is stored in browser localStorage.
  • Gmail labels drive the workflow — "3. to pay" means unpaid, "4: paid" means paid. The Apps Script sync reads from "3. to pay" and the mark-paid workflow moves emails to "4: paid".
  • Deduplication — The sync script uses the Gmail Message ID to prevent the same email from creating multiple Airtable records.
  • Inline editing — Empty fields (Amount, Payee, Ref, Due Date) can be clicked on directly in the dashboard table to enter values. Each edit is saved to Airtable immediately.
  • Transaction matching — The AI matching compares invoices against the Airtable Transactions table using exact amount + date proximity + keyword matching. Matches can be Approved (triggers mark-paid) or Rejected (saved in Airtable).

⚙ 9. Google Apps Script Setup

How the Gmail-to-Airtable Sync Works
  1. The syncGmailToAirtable function runs on a 15-minute time trigger, scanning Gmail for threads with the "3. to pay" label.
  2. For each email thread, the script extracts the subject, sender, date, Gmail message ID and thread ID.
  3. Before creating a new record, it checks Airtable for an existing record with the same Gmail Message ID to prevent duplicates.
  4. New invoices are created in the "Dashboard Invoices" table with Status set to "Unpaid" and the extracted email details.
  5. The sync can also be triggered manually from the dashboard by clicking the "Refresh from Gmail" link, which calls the Apps Script web app endpoint.
Required Script Properties

The following must be set in the Apps Script project under Project Settings > Script Properties:

PropertyValueDescription
AIRTABLE_PATpat...Airtable Personal Access Token with read/write access to the base
AIRTABLE_BASEappnqjDpqDniH3IRlThe Airtable base ID
AIRTABLE_TABLEtblkOTKIG2Tyiy9aMThe "Dashboard Invoices" table ID
Setting Up the Time Trigger
  1. Open the Apps Script project in the Google Apps Script editor.
  2. Go to Triggers (clock icon in the left sidebar).
  3. Click + Add Trigger.
  4. Set the function to syncGmailToAirtable, event source to "Time-driven", type to "Minutes timer", interval to "Every 15 minutes".
  5. Click Save and authorise the required permissions when prompted.
⚠️ If the Apps Script quota is exceeded (e.g. too many API calls), syncing pauses until the next quota reset. Check the Apps Script dashboard for execution logs.

🔧 10. Troubleshooting

Invoice email not appearing in the dashboard
Check that the email has the "3. to pay" Gmail label applied. Verify the Apps Script 15-minute trigger is active in the Apps Script dashboard. Try clicking "Refresh from Gmail" on the dashboard to trigger a manual sync. Check the Apps Script execution logs for errors.
Edits not saving
All edits save directly to Airtable via API. Check your internet connection. Open the browser developer console (F12) and look for API error messages. Verify the Airtable PAT has not expired.
No transaction match suggestions appearing
The AI matching requires an Amount to be entered on the invoice. If the amount is filled in, check that there are bank transactions in the Airtable Transactions table that haven't already been linked to other invoices.
Gmail label not moving when marking as paid
The label move is handled by the Apps Script proxy. Check that the Gmail Message ID is correctly stored on the Airtable record. Verify the Apps Script web app is deployed and accessible. Check execution logs for errors.
KPI cards showing stale data
Refresh the dashboard page. If data remains stale, check the Airtable API connection in the browser console. Verify the Airtable PAT and base/table IDs are correct.
Duplicate invoice records in Airtable
The sync script checks Gmail Message ID to prevent duplicates. If duplicates appear, check if the same email has been processed by multiple triggers. Delete the duplicate directly in Airtable.
