SOP Update Process Map

How SOPs are versioned, updated, and cascaded to linked skills

1 Detection: Identifying Stale SOPs
Trigger: SOP Needs Update
Three possible triggers:
1. User clicks "Flag for Update" on a workflow
2. Scheduled drift monitor detects changes
3. Workflow steps are edited after SOP was generated
→
Set Status: "Needs Update"
Airtable field "SOP Status" set to "Needs Update". Warning badge appears on the workflow card. Sidebar badge count increments.
→
Airtable Fields Updated
SOP Status: "Needs Update"
SOP Version: (unchanged)
SOP Last Updated: (unchanged)
Skill Status: "Stale"
Drift Monitor (Scheduled Skill): The existing "Drift Monitor" scheduled skill can be extended to compare workflow step timestamps against SOP generation date. If steps were modified after the SOP was last generated, it auto-flags the SOP as needing update.
2 Regeneration: Updating the SOP
User Clicks "Regenerate SOP"
Button appears on flagged workflows. Opens the SOP generator with pre-populated workflow data. User can review before generating.
→
AI Generates Updated SOP
Same SOP generation process as initial creation. Uses current workflow steps, descriptions, and metadata. Produces HTML output.
→
User Reviews SOP
Preview shown in modal. User can approve or request changes. No auto-publish without review.
↓
Overwrite Drive Files
HTML file replaced in existing folder. Google Doc updated (or recreated). Same folder URL preserved. No duplicate files.
→
Version Incremented
SOP Status: "Current"
SOP Version: v1 → v2
SOP Last Updated: now
SOP Change Log: appended
→
Cascade: Flag Linked Skill
If a skill was generated from this SOP, its status changes to "Needs Re-generation". Warning badge appears in Skills Library.
3 Skill Sync: Updating Linked Skills
Skill Flagged: "Needs Re-generation"
Warning badge in Skills Library. "Regenerate Skill" button appears on the skill card and in the Automation section.
→
User Clicks "Regenerate Skill"
Takes the updated SOP content as input. Re-runs the skill generation AI to produce updated skill definition (instructions, command, tags).
→
Skill Definition Updated
Skill Definition field in Airtable overwritten with new JSON. Skill version incremented. Old definition not preserved (SOP is the source of truth).
↓
Sync Complete
SOP Status: "Current"
Skill Status: "Active"
Both version numbers aligned
Warning badges cleared
→
Skills Library Updated
Next time user opens Skills Library, the updated skill definition loads from Airtable. No stale cache (fetched fresh each time).
4 Airtable Data Model
New fields on Systemisation Workflows table:

SOP Status (Single Select): "Not Started" | "Current" | "Needs Update"
SOP Version (Number): Integer starting at 1, incremented on each regeneration
SOP Last Updated (Date): Timestamp of last SOP generation/regeneration
SOP Change Log (Long Text): Append-only log. Each entry: "v2 | 2026-05-19 | Reason: Steps updated"
Skill Status (Single Select): "Not Created" | "Active" | "Needs Re-generation" | "Stale"
Skill Version (Number): Integer, incremented when skill is regenerated from updated SOP
5 Edge Cases and Rules
No Skill Exists Yet
If SOP is updated but no skill was ever created, skip Phase 3 entirely. Only the SOP version increments.
User Edits Steps After SOP
Any change to workflow steps (add, edit, delete, reorder) after SOP generation auto-flags the SOP as "Needs Update".
Multiple Skills from One SOP
If one workflow has multiple skills, ALL linked skills get flagged when the SOP is updated. Each must be regenerated individually.
Drive Folder Preserved
Regeneration overwrites files in the SAME Drive folder. The folder URL never changes. Bookmarks and links remain valid.
Manual SOP Edits
If someone edits the Google Doc directly, the system cannot detect this. Only changes via the platform trigger the cascade. Document this limitation.
Version History
The SOP Change Log is append-only. Previous SOP versions are NOT stored (the Drive file is overwritten). If version history is needed later, Google Drive's own version history provides it.
