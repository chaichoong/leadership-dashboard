🎯 1. Purpose & Architecture

The Leadership Dashboard is a single-page HTML application hosted on GitHub Pages that provides a real-time operational view of the property portfolio. All data is fetched at runtime from Airtable using a Personal Access Token (PAT) stored in the browser.

The dashboard is the central hub for daily operations and is the entry point for all other pages in the ecosystem.

🔒 2. Authentication

The dashboard uses a PIN-based authentication screen on first load. The PIN is the Airtable Personal Access Token (PAT) for the operational base. Once entered, it is stored in localStorage so you are not prompted again on the same browser.

Login Flow
  1. Navigate to the dashboard URL. The auth screen appears if no token is found in localStorage.
  2. Enter the Airtable PAT in the input field and click Access Dashboard.
  3. The system validates the token by making a test API call. If valid, the dashboard loads.
  4. If the token is rejected, an error message appears. Check the token has not expired and has the correct scopes.
⚠️ The PAT requires read and write scopes on the Airtable base. A read-only token will allow the dashboard to load but will prevent status updates (Mark as Paid, CFV actions, etc.).
ℹ️ To log out or switch tokens, open the browser console and run localStorage.clear(), then refresh the page.

📈 4. KPI Cards & Sections

The overview page is organised into four sections, each containing a grid of KPI cards. Cards expand on click to reveal a detailed breakdown. All values are fetched live from Airtable on page load and on each smart refresh cycle.

1

Financial Overview

Income & expenditure

Tracks current-month income, expenditure, and net position against budget. Cards include total rent collected, outstanding invoices, and month-to-date spend by category. Click any card to expand the per-property or per-supplier breakdown.

2

Portfolio Overview

Properties & occupancy

Displays total properties, current occupancy rate, void units, and units in the pipeline. Progress bars show occupancy against target. A red or amber card indicates occupancy has fallen below the threshold configured in Airtable.

3

Tenancy Metrics

Tenants & renewals

Shows active tenancies, tenancies expiring within 30 / 60 / 90 days, and tenancies currently in arrears. Expand any card to see the specific tenants and properties affected.

4

31-Day Operational Metrics

Rolling 31-day window

A rolling view of the past 31 days covering maintenance jobs raised, completed, and overdue; invoices processed; and CFVs opened and resolved. This section gives a quick health-check on operational throughput.

KPI Card Colour Coding
Green — On target or above Amber — Below target, monitor Red — Critical, action required Blue — Informational

💸 5. Cash Flow Forecast

The 31-Day Cash Flow Forecast section projects daily inflows and outflows for the next 31 days based on scheduled rent receipts, expected invoices, and known recurring costs from Airtable.

Reading the Forecast
ColumnDescription
DateCalendar day. Weekend rows are shaded grey.
OpeningBalance at the start of the day.
InExpected inflows (rent payments, refunds, etc.).
OutScheduled outflows (invoices, recurring costs).
NetIn minus Out for the day.
ClosingProjected balance at end of day. Red if negative.
💡 Click any forecast row to expand it and see the individual transactions contributing to that day's inflows and outflows.
⚠️ The forecast is based on expected transactions. Actual bank movements are confirmed separately through the daily reconciliation. Use the Balance Calculator to stress-test scenarios.

⚒ 6. Balance Calculator

The Balance Calculator is a collapsible tool at the top of the overview page. It allows you to select a subset of forecast transactions and calculate the projected closing balance given a known opening balance.

How to Use
  1. Click Balance Calculator to expand the panel.
  2. Enter your current bank balance in the Opening Balance field.
  3. Use the filter buttons to select Inflows Only, Outflows Only, or Select All.
  4. Tick or untick individual transactions in the list to include or exclude them from the calculation.
  5. The Closing Balance updates in real time as you make selections.
ℹ️ Use the Balance Calculator to answer questions like “If I pay all outstanding invoices this week, what will my balance be on Friday?” before committing to payments.

🤖 7. AI Analysis & Commentary

The AI Analysis & Commentary section at the bottom of the overview page generates a natural-language summary of the current operational state. It highlights key risks, positive trends, and recommended actions based on the live data.

The commentary is regenerated on each page load and smart refresh. It considers:

  • Cash flow position and any projected negative balance days
  • Open CFVs and their days overdue
  • Invoices awaiting approval or overdue for payment
  • Tenancies expiring or in arrears
  • Maintenance jobs overdue
💡 The AI commentary is a summary aid, not a replacement for reviewing the underlying data. Always verify flagged items directly in Airtable or the relevant dashboard tab before taking action.

🔄 8. Smart Refresh

The dashboard uses a smart refresh system that automatically reloads all data every 15 minutes. The refresh is deferred if the user is actively interacting with the page to avoid interruptions.

Green dot — Live and refreshing normally

The last updated timestamp in the header shows when data was most recently fetched. To force an immediate refresh, click the browser reload button or navigate away and back.

ℹ️ The 15-minute interval is hardcoded in dashboard.html as REFRESH_INTERVAL = 15 * 60 * 1000. To change it, update this constant and redeploy.

🔗 9. Sharing

A Share button is fixed in the top-right corner of the dashboard. Clicking it copies the current page URL to the clipboard and shows a confirmation toast.

⚠️ Anyone with the URL can see the auth screen, but they will need the Airtable PAT to access data. Do not share the PAT. If a shared user needs read-only access, create a separate read-only Airtable token for them.

🔧 10. Troubleshooting

Dashboard shows auth screen despite entering the correct PIN
The PAT may have expired. Log in to Airtable, navigate to Account → Developer Hub, and verify the token is active. Generate a new token if needed and re-enter it on the auth screen.
KPI cards show “Loading” indefinitely
Open the browser developer tools (F12) and check the Console and Network tabs for failed API requests. Common causes: expired PAT, Airtable API rate limit (429), or a network connectivity issue.
Cash flow chart is blank
Verify the Chart.js CDN link is reachable. If offline, the chart will not render. The data table below the chart is still populated from Airtable data. Check the browser console for script load errors.
AI commentary not appearing
The AI commentary relies on a successful data fetch. If any upstream section fails to load, the commentary may be empty or show an error message. Resolve the data fetch issue first, then refresh.
Sidebar not visible on mobile
Tap the hamburger icon (☰) in the top-left corner to open the sidebar. On screens narrower than 768px the sidebar is hidden by default and must be toggled manually.
Data looks stale after a known Airtable update
Wait up to 15 minutes for the next smart refresh cycle, or force a reload with Cmd+Shift+R (Mac) / Ctrl+Shift+R (Windows) to bypass the browser cache.
