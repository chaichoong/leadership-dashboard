🎯 1. Purpose & Overview

💡 What It Does

The Business Plan Builder is an AI-powered conversational tool that guides you through creating a complete business plan. It uses the Anthropic Claude API to have a natural chat conversation, walking you through 10 key sections one at a time.

You talk to the AI, answer its questions, and it builds a professional business plan with rich visual output — KPI cards, tabbed sections, timeline bars, and collapsible accordions. No templates to fill in, no blank pages to stare at. Just a conversation.

💬
Think of it like this: Instead of writing a business plan from scratch, you have a chat with an AI advisor who asks you the right questions and turns your answers into a polished plan.

🏗 2. System Architecture

🌐 Component Overview

The Business Plan Builder is a client-side HTML/JavaScript application that connects to the Anthropic Claude API through a local Python proxy server. The proxy is needed because browsers block direct API calls to external services (a security feature called CORS).

💻
Browser UI
index.html
Chat + Plan views
➡️
🐍
Proxy Server
serve.py
Port 8095
➡️
🤖
Claude API
api.anthropic.com
AI responses
ℹ️
Why the proxy? Browsers won't let a web page call api.anthropic.com directly (CORS policy). The proxy runs on localhost (same origin as the page), so the browser allows it. It's a simple pass-through — no data is stored.

⚙️ 3. Initial Setup

1

Initial Setup

First-time & every session

Start the Proxy Server

  1. Open a terminal window on your computer.
  2. Navigate to the business plan builder folder: cd os/business-plan-builder
  3. Run the proxy server: python3 serve.py
  4. You should see a message saying the server is running on port 8095.
  5. Leave this terminal window open — closing it stops the server.

Open the Application

  1. Open your browser and go to http://localhost:8095
  2. The Setup Screen appears asking for your API key and model choice.
  3. Enter your Anthropic API key (starts with sk-ant-).
  4. Select a model from the dropdown (e.g. Claude Sonnet, Claude Opus).
  5. Click Start Building to enter the chat view.
⚠️
API key stays local: Your API key is only sent from your browser to the local proxy server, which forwards it to Anthropic. It is never stored on disk or sent anywhere else.

💬 4. Building the Plan

2

Building the Plan

Core workflow

The Chat View

After setup, you enter a chat view with two main areas:

  • Sidebar (left): Shows all 10 business plan sections. The current section is highlighted. Completed sections get a green tick.
  • Chat area (right): A messaging interface where you talk to the AI. Type your messages at the bottom and press Enter or click Send.

How the Conversation Works

  1. The AI starts by introducing itself and asking about your business idea for the Executive Summary section.
  2. Type your answer and send it. The AI will ask follow-up questions to get more detail.
  3. After enough information is gathered, the AI saves that section and automatically moves to the next one.
  4. A green "Section captured" message appears in the chat when a section is complete.
  5. This process repeats for all 10 sections until the entire plan is built.
💡
Be detailed: The more information you give the AI, the better your plan will be. Don't just give one-word answers — explain your thinking. The AI uses everything you say to write each section.

📊 5. Viewing the Plan Output

3

Viewing the Plan Output

Visual output

Opening the Plan View

  1. Once at least one section is complete, the "View Plan" button appears in the sidebar footer.
  2. Click it to open a full-screen overlay showing your business plan.
  3. The plan opens with a header, KPI summary cards, and tabbed navigation for each section.

Plan View Features

📈 KPI Cards

A row of 4 summary cards at the top showing key metrics extracted from your plan (e.g. target market size, revenue projections, team size).

📑 Tabbed Navigation

Sticky tab bar to jump between sections. Tabs are colour-coded by group: Foundation (blue), Commercial (green), Execution (amber).

📄 Section Cards

Each section displays its content in styled cards with headers, bullet points, and highlight boxes for key figures.

📅 Timeline & Accordions

The Implementation Timeline shows visual progress bars. Some sections use collapsible accordions to keep the view tidy.

Exporting / Printing

  1. Click the "Export PDF" button in the plan view toolbar.
  2. Your browser's print dialog opens with the plan formatted for printing.
  3. Choose Save as PDF or send directly to a printer.
🖨️
Print styling: The plan view includes dedicated print CSS that hides the toolbar, adjusts colours for paper, and ensures page breaks fall in sensible places.

✏️ 6. Reviewing & Editing Sections

4

Reviewing & Editing Sections

Refine your plan

View Discussion

After a section is complete, you can review the conversation that built it:

  1. Click a completed section (green tick) in the sidebar.
  2. A Review Panel opens showing the section data and the full AI conversation for that section.
  3. Scroll through the discussion to see exactly what questions the AI asked and what you answered.

Edit Section

Want to change something? You can send follow-up edits to any completed section:

  1. In the Review Panel, find the Edit Section area at the bottom.
  2. Type your edit instruction (e.g. "Change the revenue target to £500k" or "Add a section about partnerships").
  3. Click Send Edit. The AI processes your change and updates the section.
  4. The updated content appears immediately in the review panel and plan view.
✅
Non-destructive edits: Editing a section doesn't lose any data. The AI takes the existing section content, applies your change, and saves the updated version. You can edit as many times as you like.

💾 8. Data Persistence & Storage

🗄️ How Your Data Is Saved

The Business Plan Builder saves all your progress to your browser's localStorage. This means your data stays on your computer and persists between browser sessions (unlike sessionStorage, which clears when you close the tab).

ItemDetails
Storage Key od-bplan-v4
What's Saved All section data, chat history, section completion status, KPI values
Where It Lives Browser localStorage (on your machine only)
Persistence Survives browser restarts. Only cleared if you manually clear browser data or use the app's reset function.
⚠️
Clearing browser data will delete your plan. If you clear your browser's cache/storage, or use a "Clear All Data" function in the browser, your saved business plan will be lost. Export to PDF before clearing.
💡
Different browsers = different data. localStorage is per-browser. A plan saved in Chrome won't appear in Safari. Stick to one browser for your planning session.

🐍 9. Proxy Server (serve.py)

5

Technical Architecture — Proxy Server

Technical

What It Does

The serve.py file is a small Python HTTP server that does two things:

  • Serves the HTML file: When you visit http://localhost:8095, it serves index.html.
  • Proxies API calls: When the browser sends a request to /api/messages, the proxy forwards it to https://api.anthropic.com/v1/messages and returns the response.

Why It's Needed

Browsers enforce a security rule called CORS (Cross-Origin Resource Sharing). This prevents web pages from making API calls to different domains. The proxy runs on localhost (same origin as the page), so the browser allows it.

Request Flow

💻 Browser User sends message
➡️
🌐 localhost:8095 /api/messages
➡️
🐍 serve.py Forwards request
➡️
🤖 Claude API Returns response

Running the Server

CommandDescription
python3 serve.py Start the proxy server on port 8095
Ctrl + C Stop the server
lsof -i :8095 Check if anything is already using port 8095
🚨
Server must be running: If the proxy server isn't running, the app will show a "Failed to fetch" error when you try to chat. Always start serve.py before using the app.

🔧 10. Troubleshooting

🛠️ Common Issues & Fixes
❌ "Failed to fetch" or CORS error when chatting
Cause: The proxy server (serve.py) is not running.
Fix: Open a terminal, navigate to the os/business-plan-builder folder, and run python3 serve.py. Make sure it says it's listening on port 8095 before trying again.
❌ "messages: at least 1 message is required" error
Cause: You tried to send an empty first message, or the message array wasn't built correctly.
Fix: Make sure you type something in the chat box before pressing Send. If the error persists, refresh the page and try again.
❌ API key error (401 Unauthorized)
Cause: The API key is invalid, expired, or wasn't entered correctly.
Fix: Go back to the setup screen and re-enter your Anthropic API key. Make sure it starts with sk-ant- and has no extra spaces. Check your Anthropic console to confirm the key is active.
❌ My plan has disappeared
Cause: Browser localStorage was cleared, or you're using a different browser/profile.
Fix: Unfortunately, if localStorage was cleared, the data is gone. To prevent this: (1) always export your plan to PDF when it's finished, (2) don't clear browser data while working on a plan, (3) use the same browser and profile each time.
❌ Port 8095 already in use
Cause: Another instance of serve.py is already running, or another app is using port 8095.
Fix: Find the process with lsof -i :8095 to get the PID, then kill <PID>. Then restart the server.
❌ The AI seems stuck or the typing indicator won't stop
Cause: The API request may have timed out or the response was too large.
Fix: Wait 30 seconds. If still stuck, refresh the page. Your progress is saved in localStorage, so you won't lose completed sections.
📞 Getting Help

If you run into an issue not listed above:

  • Check the browser's Developer Console (press F12 → Console tab) for error messages.
  • Check the terminal running serve.py for any Python error messages.
  • Make sure you have a working internet connection (required for the Claude API).
