📋 1. Overview & Purpose

The Task and Project Management OS is a self-contained web application that provides the operations team with a unified view of all tasks and projects stored in Airtable. It is designed for daily operational use by Kevin Brittain, Mica Albovias, and Ericamae Atenta, and provides a separate mobile-optimised view for contractors (Gary, Roy, Rob).

The system fetches all non-completed, non-someday tasks from Airtable on every load and presents them across six specialist views. Tasks can be updated in real-time — status changes and due-date adjustments write directly back to Airtable via the REST API. No separate backend is required.

Key capabilities

  • Six views: My Day, Task List, Kanban (by week), Projects, Capacity, and Recurring
  • Smart Daily Planner that slots tasks around calendar events and personal routines
  • Google Calendar integration via a Google Apps Script proxy
  • Automatic rescheduling of Kevin's overdue tasks to today on every load
  • Team and personal identity switcher (each user sees their own tasks by default)
  • Bulk operations: mark multiple tasks complete or reschedule them in one action
  • Drag-and-drop Kanban board for scheduling project tasks across calendar weeks
  • Separate contractor jobs page (jobs.html) optimised for mobile use
🔑 2. How to Access
1

First-time setup

One-off
  1. Open os/tasks/index.html in your browser (or via the OS Hub iframe).
  2. If prompted, enter your Airtable Personal Access Token (starting with pat_) in the auth bar at the top and click Connect. The token is stored in localStorage under the key _dlr_pat and will not be asked for again on the same device.
  3. The identity overlay appears. Select your name (Kevin Brittain, Mica Albovias, or Ericamae Atenta). Your choice is saved to localStorage under _task_user. The system defaults to showing your tasks in the team filter.
  4. Data loads automatically. The top bar shows the sync timestamp once loading is complete.
ℹ If the app is loaded inside the main platform iframe, it automatically inherits the parent window's PAT via window.parent.PAT and skips the auth bar entirely.
2

Returning visits

Daily use
  1. The page loads with your saved identity and PAT. No login is needed.
  2. Data is fetched from Airtable in parallel with calendar events on every load.
  3. Kevin's overdue tasks are automatically rescheduled to today in the background.
  4. To reload manually at any time, click Refresh in the top-right corner.
  5. To switch user, click Switch next to your name badge in the top bar. This clears the saved identity and shows the selector again.
📊 3. Task Views

The six views are accessible via the tab bar below the KPI cards. Each view serves a distinct purpose and may show a different subset of tasks.

☀ My Day

The default view. Shows today's tasks and overdue tasks for the active team member. Builds a visual day timeline from 08:00 to 18:00 (configurable) by slotting tasks into gaps between calendar events and routine blocks. Includes KPI row, capacity bar, deferred-task warnings, and inline status dropdowns.

✔ Tasks with no time estimate are still shown in the "Also Today" section below the timeline.
📋 Task List

A paginated table of all non-someday, non-Project-priority tasks. Each row shows task name, due date, status badge, assignee avatar, priority, time estimate, business, and recurring flag. Clicking any row expands it in-place to show description, metadata, attachments, a status dropdown, and an "Open in Airtable" link. Supports bulk selection.

ℹ The Task List deliberately excludes Project-priority tasks; those are managed in the Kanban view.
📌 Kanban

A four-column week-by-week board showing only Project-priority tasks for the selected month. Columns are Week 1 (days 1–7), Week 2 (8–14), Week 3 (15–21), Week 4 (22–end). Each column header shows the date range and total hours allocated. Cards are draggable — dropping a card onto a different column updates the task's due date to the first day of that week in Airtable. Month navigation uses the Previous / Next buttons.

📁 Projects

A card grid of all projects from the Projects Airtable table. Each card shows project name, status badge, progress bar (completed tasks / total tasks), overdue count, and date range. Clicking a card expands it to show the Deliverable, Definition of Done, and up to 20 linked tasks. Filter by Active, Completed, or All. Projects sorted: active first, then by completion percentage descending.

📈 Capacity

Per-person utilisation cards showing total hours allocated vs. the 40 hr/week capacity constant. Colour coding: green below 70%, amber 70–90%, red above 90%. Each card breaks down hours by status and by priority. When "All" is selected in the team switcher, a Team Overview card appears across the top spanning the full grid width.

🔄 Recurring

All tasks where the Recurring field or Recurring? flag is set. Groups tasks by frequency: Daily, Weekly, Fortnightly, Monthly, Quarterly, Bi-Annually, Annually, Flagged. Shows summary KPIs (total recurring, hours per cycle, completed, overdue). Intended to identify candidates for AI agent automation.

ℹ All views respect the Team Switcher at the top. Selecting a person's name filters every view to only their tasks. Selecting All shows the whole team.

✏ 5. Creating & Editing Tasks
Creating tasks

Tasks are created directly in Airtable (table tblqB8b22hKBL4PF1). The Task Manager OS does not provide a create-task form — it is a read and update interface only.

⚠ After creating a task in Airtable, click Refresh in the top bar to pull the latest data into the OS page.
Editing task status (in-page)

Status can be changed without leaving the page in three ways:

  1. In My Day / timeline: click the Status button on any task card to open a dropdown with all statuses (Today, Upcoming, To do, Approval, Completed, Overdue). Or click Complete to jump straight to Completed.
  2. In Task List: expand a task row, then use the Change Status dropdown in the detail panel. The page re-renders immediately after the Airtable PATCH request succeeds.
  3. In My Day / card list: click the Status button on a card card, or click Done to mark Completed directly.

All status changes call patchTask(taskId, { [F.status]: { name: newStatus } }) which sends a PATCH request to https://api.airtable.com/v0/{BASE_ID}/{TASKS_TABLE}/{taskId} with typecast: true.

Editing due dates (in-page)

Due dates can be updated in these ways:

  • Drag a Kanban card to a different week column. The due date is set to the first day of the target week.
  • Click Defer to tomorrow in the Plan My Day overlay. This sets the due date to tomorrow's date.
  • The bulk Reschedule Selected button sets the due date to tomorrow for all ticked tasks.

All due-date changes call updateTaskDueDate(taskId, newDate) which sends a PATCH to the same endpoint with field fld7XP8w8kbxfETV4.

Opening in Airtable

In the Task List expanded row, an Open in Airtable button opens the record directly at https://airtable.com/{BASE_ID}/{TASKS_TABLE}/{recordId} in a new tab. Use this for full editing (all fields, attachments, linked records).

🗓 6. Smart Daily Planner
How the planner works

The Smart Daily Planner is triggered when the My Day view loads and also when Plan My Day is clicked. It calls buildDayTimeline(todayTasks, overdueTasks, calEvents, routineBlocks) to construct a minute-by-minute schedule for the working day.

Algorithm (in order)

  1. Place calendar events as fixed blocks. All-day events are shown separately and do not block slots. Timed events are placed by parsing their start/end times to minutes since midnight.
  2. Place routine blocks using placeRoutineBlocks(). The "Run" block is placed first (preferred afternoon = 14:00). Lunch is placed next with the rule "must finish 2 hrs before Run". "Get Ready" is placed at the start of the working day. Any custom blocks are placed in order.
  3. Find free slots using findFreeSlots(). A slot must be at least 15 minutes long to be used.
  4. Schedule tasks into free slots in priority order (Urgent first, then High, then Not Urgent). Overdue tasks are included before today's tasks. Each task uses its timeDur field (seconds, converted to minutes) if set; otherwise it falls back to timeEst name (e.g. "1 hr" = 60 min); otherwise defaults to 30 minutes.
  5. Deferred tasks are those that could not fit in any remaining slot. They appear in the "Won't Fit Today" warning section.
Routine configuration

Click Configure Routine in the My Day view to open the routine editor panel. Settings are saved to localStorage and persist between sessions.

SettinglocalStorage keyDefault value
Routine blocks (name, duration, preferred time, rule) _task_routine Get Ready (30 min, morning), Lunch (30 min, midday, before-run-2h), Run (60 min, afternoon)
Working hours config (start, end, max task hours) _task_workconfig Start: 8:00, End: 18:00, Max task hours: 6
ℹ The available rule options for a routine block are None and 2h before Run. Click Reset to Defaults to remove saved settings from localStorage and restore the three default blocks.
Plan My Day overlay

Clicking Plan My Day opens a modal overlay showing:

  • Two KPI tiles: total task time vs. total free time
  • If overcommitted: a warning with the excess amount, and a Defer to tomorrow button for each suggested deferral (lowest-priority tasks first)
  • If capacity is OK: a confirmation that all tasks fit
  • A list of scheduled tasks with their proposed time slots (start time — end time, duration)

Clicking outside the modal or pressing Close dismisses it.

Auto-reschedule (Kevin only)

Every time data loads, autoRescheduleOverdue() runs automatically for Kevin's account. It finds all tasks where:

  • Assignee email = kevin@runpreneur.org.uk
  • Due date is before today
  • Status is not "Completed"
  • someDay field is not set

For each matching task it sets the due date to today. If the task has no origDueDate yet, it also saves the original due date to the fldS2OR7oQKC7T1to field. Patches are sent in batches of 10 with a 250 ms delay between batches to avoid rate limits.

⚠ This runs silently on load. The My Day view shows a notice confirming how many tasks were rescheduled. It does not run for Mica or Ericamae.
📅 7. Calendar Integration
How Google Calendar data is fetched

The page calls fetchCalendarEvents() on every data load. This makes a GET request to the Google Apps Script proxy URL configured in the constant GCAL_SCRIPT_URL at the top of the script, passing today's date as a query parameter (?date=YYYY-MM-DD).

The proxy (deployed as a Google Apps Script web app) returns a JSON object with an events array. Each event must have: summary, start, end, optionally location, and allDay. Start and end can be either HH:MM or an ISO datetime string.

⚠ If GCAL_SCRIPT_URL is empty, calendar fetching is skipped silently and a "Calendar: not connected" note appears in the My Day action bar. The rest of the page still works normally.
Calendar events in My Day

Once fetched, calendar events appear in the My Day timeline as blue "CALENDAR" blocks. All-day events appear in a separate "All-Day Events" section above the timeline, not as time-slot blocks. The count of calendar events for today is shown in the My Day KPI row and in the page subtitle.

✔ Calendar events are treated as immovable fixed blocks. The planner will not place tasks or routine blocks during calendar event time slots.
👷 8. Contractor Jobs View
Purpose and access

The file os/tasks/jobs.html is a separate, mobile-optimised page for contractors. It shows each contractor only their own active, non-completed tasks. The interface is minimal and touch-friendly, with large tap targets (minimum 44 px height) and no complex views.

Contractors: Gary, Roy, Rob. The same Airtable base and tasks table are used (appnqjDpqDniH3IRl, tblqB8b22hKBL4PF1). The contractor's name is used as the filter value for the Assignee field in the Airtable formula: AND({Status}!="Completed", {Assignee}="{name}").

1

First-time setup (contractor)

  1. Open os/tasks/jobs.html on your mobile browser.
  2. If the auth overlay appears, enter the Airtable PAT and tap Connect. The PAT is saved to localStorage (_dlr_pat).
  3. The name picker shows: tap your name (Gary, Roy, or Rob). Your selection is saved to localStorage (_task_contractor).
  4. Your active jobs load in priority and status order.
Job card interactions

Each job appears as a card with the task name, due date, status badge, priority badge, and business badge. Tapping a card expands it to show:

  • Description (if set)
  • Notes (existing notes from Airtable)
  • Time estimate (if set)
  • Attachments as tappable links (images show a thumbnail)
  • Two action buttons: Mark Complete and Add Note
✔ Only one card can be expanded at a time. Tapping another card collapses the previous one and scrolls the new one into view.

🔍 10. Filtering, Sorting & Search
Task List filters & sort

All controls live in the .controls bar above the task table. Every change immediately triggers renderTasks() via oninput or onchange events and resets to page 1.

A dropdown in the Kanban controls bar lists all unique project names found across tasks with priority "Project". Selecting a name sets kanbanProjectFilter and re-renders the board. Tasks with no matching project ID or name are excluded.

The Recurring view groups tasks by their recurring frequency field value. The display order is hard-coded as: Daily, Weekly, Fortnightly, Monthly, Quarterly, Bi-Annually, Annually, Flagged. "Flagged" captures tasks where the Recurring? checkbox is true but the recurring text field is empty.

🔧 12. Troubleshooting
Page shows "Failed to load data" or "Invalid token. Please re-enter your PAT"
Your Airtable PAT has expired or is incorrect. The auth bar reappears automatically when the API returns a 401 or 403. Enter a valid PAT and click Connect. This clears the stored token and retries.
Page asks for a PAT on every load even though I already entered it
The page is running in a context where localStorage is blocked (private/incognito mode, or an iframe with restricted storage). Use a regular browser tab, or ensure the platform shell is loaded first so the child iframe inherits the parent's PAT via window.parent.PAT.
Tasks I just created in Airtable are not showing
Click Refresh in the top bar. The page only fetches data on load and on explicit refresh — it does not poll Airtable in the background.
The Kanban board is empty
The Kanban view shows only tasks with priority "Project". Check that your tasks are assigned the "Project" priority in Airtable. Also check the month navigation — tasks are placed by their due date, so a different month may be selected.
My Day timeline shows no calendar events
Check that GCAL_SCRIPT_URL is set in the script constants and that the Google Apps Script proxy is deployed and accessible. If the constant is empty, a "Calendar: not connected" note appears in the My Day action bar. Calendar fetch errors are logged to the browser console.
Auto-reschedule ran but some overdue tasks were not rescheduled
Auto-reschedule only runs for Kevin (key "kevin"). It skips tasks where someDay is true. Check that the signed-in user is Kevin and that the task's someDay field is not checked in Airtable.
Status change button does not update; page re-renders but old status shows
The PATCH request may have failed silently. Open the browser console and check for errors. Common cause: the PAT lacks write permission on the tasks table. Ensure the PAT has "create records, update records" scope on the base.
Contractor jobs page (jobs.html) shows "Error loading" with an API error message
The error message is displayed inside the task list area with a Retry button. Common causes: expired PAT, network failure, or invalid contractor name (name must match the Assignee field in Airtable exactly). Tap Retry to try again, or Switch user to re-select the contractor.
Capacity view shows 0 hrs for a team member who has tasks
Capacity calculates hours using timeDur (seconds field) first, then timeEst (select field). If neither is set, the task contributes 0 minutes. Ensure tasks have a Time Estimate or Time Duration set in Airtable.
Drag-and-drop on the Kanban board does not update the due date
The drop handler calls updateTaskDueDate(draggedTaskId, weekStartDate) which requires a successful PATCH. If the API call fails, the card visually moves but reverts on the next render. Check the browser console for errors and verify PAT write permissions.
