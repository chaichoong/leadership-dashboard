🏢 The Departments

Operations Director is organised into departments, just like a real company. Each department has sections that handle specific jobs. Click a department to see its sections.

🧠 The Brain

Every section reads from and writes to the same central database. Think of it as a giant filing cabinet that remembers everything.

🗄️

Airtable Database

Airtable is like a super-powered spreadsheet. It stores all the data for the business in separate tables (like different drawers in a filing cabinet). When Kevin logs in, the app pulls data from these tables and shows it on screen.

🏥 The Health Monitor

Every section has a built-in health check, like a doctor's checkup for the system. It tells Kevin if everything is working or if something needs attention.

Green = Everything is working. Data is fresh (updated less than 5 minutes ago) and all checks passed.
Amber = Something needs attention. Data might be slightly old (5 to 30 minutes) or a check found a warning.
Red = Something is wrong. Data is stale (over 30 minutes old) or a check failed.
Grey = Not checked yet. The section has not been opened since login.
Where to see it: Every section in the sidebar has a small coloured dot next to its name. That dot is the health status. Inside each section, there is a "health bar" at the top that shows exactly what was checked and whether it passed or failed. Click "Re-run" to run the checks again.

How it works: When a section loads data, it runs "passive" checks automatically (things like "is the data fresh?"). There are also "active" checks that do heavier work (like testing if the database connection is alive). Active checks only run when Kevin clicks the "Re-run" button.

The sidebar rollup: Each department heading also has a dot. It shows the worst status of any section inside it. So if one section in Finance is red, the Finance department dot turns red too. This lets Kevin spot problems at a glance without opening every section.
📚 The Guidebooks (SOPs)

Every section has a Standard Operating Procedure (SOP). Think of it as an instruction manual that explains how to use that section step by step.

SOPs live on the "Site Map & Guides" page. Each one is a separate web page that explains:

  • What the section does and why it matters
  • Step-by-step instructions for every action
  • What the health checks look for
  • Troubleshooting tips if something goes wrong

Version tracking: Every section has a version number (like v2.31). When the code changes, the version number goes up automatically. The Site Map page compares the section version to its SOP version. If they do not match, it means the guide needs updating.

🕸️ How It All Connects

Click any section above and look at the "Connects To" tags. Here is the summary of the biggest connections.

💷 Rent comes in
Tenants pay rent → appears as a bank transaction → Reconciliation matches it to the right tenant → Income tab shows it → P&L counts it as revenue → Cash Flow forecasts future balance
🧾 Invoice arrives
Email arrives in Gmail → Gmail Script detects it → Creates record in Airtable → Invoices tab shows it → Kevin approves payment → Reconciliation matches the bank transaction → P&L counts it as an expense
🚨 Tenant misses rent
Due date passes with no payment → CFV engine detects it → Flags the tenancy → Rent Statements calculate the balance → Sidebar badge shows alert count → When tenant pays, CFV auto-returns status to normal
📋 Monthly cost due
Cost record has a due day → Costs tab checks if a matching transaction arrived → If yes, marks "In Payment" → If no, marks "Overdue" → Variance flag shows if amount differs from expected → P&L totals it
🎯 Strategy to tasks
Kevin sets quarterly objectives → Strategy page creates projects → Projects break into tasks → Tasks appear on the Tasks page → Team members get assigned → KPIs track project progress
🤖 AI Assistant
Gathers context from whatever tab Kevin is viewing → Sends it plus Kevin's question to Claude → Returns a data-aware answer → Can reference SOPs, transactions, KPIs, and more
