🎯 1. Purpose

The Site Map page serves as the central index for all pages within the Leadership Dashboard ecosystem. It provides a single point of reference for locating, sharing, and verifying the integrity of every HTML page deployed to the site.

Use this page to:

  • Find the URL for any dashboard page or SOP
  • Check the current version and last-updated date of each page
  • Share direct links with team members or contractors
  • Run an integrity check to verify all pages are loading correctly

📄 3. Version Tracking System

Each page displays its version in the header area. The versioning convention is:

  • Major version (e.g. 1.0, 2.0) — Significant restructuring or new sections added
  • Minor version (e.g. 1.1, 1.2) — Content corrections, clarifications, or small additions

Version numbers are stored in the HTML file header alongside the date and owner. When updating a page, always increment the version and update the date.

💡 The Site Map page itself tracks the version of every page. If a page is updated but the Site Map is not, the integrity check will flag the discrepancy.

🔗 4. Sharing Pages

How to Share a Dashboard Page
  1. Navigate to the Site Map page on the dashboard.
  2. Find the page you want to share in the Page Registry table.
  3. Click the page link to open it, then copy the URL from the browser address bar.
  4. Share the URL via Slack, email, or WhatsApp. The pages are publicly accessible via GitHub Pages (no login required).
⚠️ All dashboard pages are publicly accessible. Do not include sensitive data (account numbers, personal addresses) directly in the HTML. Sensitive data is fetched from Airtable at runtime and only visible to authenticated users.

✅ 5. Integrity Check

The Site Map page includes a built-in integrity check that verifies all registered pages are loading correctly. The check performs the following:

  • HTTP Status — Fetches each page URL and verifies it returns a 200 OK response.
  • Title Verification — Checks that the page title matches the expected title from the registry.
  • Version Match — Compares the version in the page header against the version recorded in the Site Map registry.

Results are displayed as a status grid:

Pass — All checks OK Warning — Version mismatch Fail — Page not loading

➕ 6. Adding a New Page

Steps to Add a New Dashboard Page
  1. Create the HTML file in the Leadership Dashboard directory using the standard CSS template (copy from any existing SOP).
  2. Include the standard header with title, subtitle, version (start at 1.0), date, and owner.
  3. Add the file to the GitHub Pages deployment (commit and push to the repository).
  4. Update the Page Registry table in this Site Map SOP with the new page details.
  5. Run the integrity check to verify the new page loads correctly.
ℹ️ Use the SOP Generator skill in Claude Code to create new SOP pages automatically. It will apply the correct CSS template and section structure.

📝 7. Updating an SOP

Steps to Update an Existing SOP
  1. Open the HTML file in a code editor or ask Claude Code to make the changes.
  2. Make the required content changes (add sections, update text, fix errors).
  3. Increment the version number in the header (e.g. 1.0 to 1.1 for minor changes).
  4. Update the date in the header to today's date.
  5. Commit and push the updated file to GitHub Pages.
  6. Update the version in the Site Map Page Registry if it has changed.
💡 Keep a changelog in the HTML comments at the top of the file. This helps track what changed between versions without needing to review git diffs.
