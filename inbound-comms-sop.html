3

Oldest First

📊 Custom Columns

This view has unique columns not found in other labels:

  • Invoice Date — When the invoice was received
  • Days Old — Age of the invoice
  • Last Activity — Most recent action taken
  • Last Update — Days since last delay email was sent

📋 Steps

  1. Select "3. to pay" from the label dropdown
  2. Review outstanding invoices sorted oldest first
  3. Click "Delay Payment" to generate an AI-drafted financial difficulty email
  4. The loading screen shows "Claude is drafting a delayed payment message..."
  5. The AI references your financial situation and avoids repeating the same wording to the same creditor
  6. Review, amend if needed, and optionally add attachments via the Add Attachment button

  8. When the invoice is eventually paid, use the Move dropdown to move to label 4 (paid)
ℹ️
No label move on send: Unlike other labels, sending a delay payment email does not move the email to another label. The email stays in To Pay because the invoice is still outstanding. The automation will detect when the invoice is reconciled.
⚠️
Delay History Tracking: The system tracks all delay emails per creditor in localStorage. The AI uses this history to avoid sending repetitive or identical delay notifications.

🤖 11. AI Features & Rules

🏷️ Label Suggestions

Model: Claude Haiku (fast, cheap)

Where: Inbox view

Classifies incoming emails and suggests the appropriate label (1-9). Learns from corrections stored in localStorage and synced to Cloudflare KV.

✉️ Response & Follow-up Drafting

Model: Claude Sonnet (quality)

Where: Labels 1, 2, and all other labels with action buttons

Drafts replies and follow-ups based on email content and your knowledge base entries. The loading message adapts to the context:

  • Label 1: "Claude is drafting your response..."
  • Label 2: "Claude is drafting your follow-up..."
  • Label 3: "Claude is drafting a delayed payment message..."
  • Label 7: "Claude is drafting your unsubscribe request..."

All draft modals support file attachments (up to 20MB total). Click "Add Attachment" to attach files before sending.

💰 Delay Payment Drafts

Model: Claude Sonnet (quality)

Where: Label 3

Drafts financial difficulty emails. Tracks delay history per creditor to avoid repetitive wording.

🚫 Unsubscribe Drafts

Model: Claude Haiku (fast)

Where: Label 7

Drafts polite unsubscribe request emails when auto-methods are not available.

🚨
CRITICAL RULE: All AI-generated emails must NOT contain em dashes (—) or en dashes (–). This rule is enforced in the AI prompts.

🔍 12. Search, Pagination & Undo

🔎 Search & Filter

A search box appears between the stats bar and the email list in all views. There are two search modes:

Local Search (“This Label”)

  • Real-time filtering with 300ms debounce as you type
  • Searches across: subject, sender name, sender email, snippet text, and full email body content (HTML is converted to plain text for searching)
  • Shows “Showing X of Y emails” when a filter is active
  • Only searches emails loaded in the current label view

Global Search (“All Emails”)

  • Click the “This Label” button next to the search box to toggle to “All Emails” mode (button turns purple)
  • Type your search term and press Enter to search
  • Uses the Gmail API server-side search — same engine as the Gmail search bar
  • Searches your entire Gmail account (all labels, inbox, sent, archived)
  • Each result shows a purple label tag indicating which of the 9 numbered labels it belongs to (or “Unlabelled”)
  • Results display in a simplified read-only view (subject, from, label, date, days old)
  • Click any row to expand and read the full thread
  • Click “All Emails” again or clear the search to return to the current label view
ℹ️
Global search limits: Results are capped at 100 matches for speed (the 100 most recent). Because it searches the entire Gmail account, results may include emails outside the 9 numbered labels — these show as “Unlabelled”.
✅
Full body search supported: Local search covers subject, sender, snippet, and full email body content. HTML emails are automatically converted to plain text for searching.

Clear button (X) resets both search modes. Pagination applies to all search results.

📄 Pagination

All views display 50 emails per page with navigation controls below the email list.

  • Previous / Next buttons for navigation
  • "Page X of Y" indicator shows current position
  • Stats and metrics always reflect ALL emails, not just the current page
  • Page resets to 1 when switching labels, refreshing, or searching

💾 13. Backup & Cloud Sync

📦 Auto-Backup System

The app automatically backs up all localStorage data after key actions:

  • Building knowledge base
  • Applying inbox labels in bulk
  • Sending emails

On app load, if the last backup was 24+ hours ago, it auto-exports a backup JSON file.

📋 What's Included in Backups

  • Knowledge base
  • AI corrections
  • Accuracy log
  • Delay payment log
  • Read newsletters
  • Unsubscribe log
  • Sender name
  • Applied today counts
  • Response amendments

⚙️ Manual Backup & Restore

Available in Settings > Backup & Restore section:

  • Export — Downloads a JSON file with all app data
  • Import — Restores from a previously exported JSON file
💡
A "Backup: Xh ago" indicator in the header shows when the last backup was created. If this indicator is missing, the auto-backup code may not have loaded.
☁️ Shared Cloud Data Store (Cloudflare KV)

Knowledge base, AI corrections, and KB build date are synced to Cloudflare KV storage for multi-device support.

🔄 How Cloud Sync Works

  1. On app load (after authentication), the app fetches cloud data from GET /shared-data
  2. If cloud data is newer than local, the cloud version replaces local data
  3. If local data is newer than cloud, local data syncs up to the cloud
  4. After building knowledge base or logging AI corrections, data syncs to cloud automatically

📊 Cloud Sync Status Indicators

Synced — data is up to date Syncing... — transfer in progress Offline — cloud unreachable
☁️
Multi-device support: Any device opening the app gets the same learned data. Failures are handled gracefully — the app falls back to localStorage silently.

🔧 17. Troubleshooting

🔴 "Too many concurrent requests"
The app processes label changes sequentially with 300ms delays to avoid Gmail rate limits. Wait a few seconds and retry. This usually happens when applying many labels at once.
🔴 Emails not loading
Check that you are signed in (click the Sign In button). The session expires periodically and needs re-authentication via Google OAuth.
🔴 AI not working / drafts not generating
Check that the Cloudflare Worker (claude-proxy.kevinbrittain.workers.dev) is running. Test by visiting the URL directly in a browser. If down, check the Cloudflare Workers dashboard.
🔴 Scheduled tasks not running
Open Claude Desktop and check the Scheduled section. Both tasks need tool permissions approved on first run. Verify they show recent successful runs at 8am, 12pm, or 4pm UK time.
🔴 Airtable tasks not being created
Verify the scheduled task inbound-comms-create-airtable-tasks is enabled and has run recently. Also check Airtable access token is still valid.
🔴 AI suggestions seem inaccurate
The AI learns from corrections. Keep correcting wrong suggestions — the accuracy should improve over time. Check the accuracy percentage in the inbox view header. If localStorage was cleared, the learning history was lost.
🔴 Cloud sync shows "Offline"
The Cloudflare Worker KV binding may not be set up correctly. Check that the SHARED_DATA namespace exists and is bound to the worker in the Cloudflare dashboard under Settings > Bindings.
🔴 Knowledge base not syncing across devices
Ensure you have built the KB at least once after the cloud sync update. The sync only happens after KB builds or AI corrections. Open the app on each device and check the cloud sync indicator in the header.
🔴 Backup not happening
Check the "Backup: Xh ago" indicator in the header. If it is missing, the auto-backup code may not have loaded. Try a manual export from Settings > Backup & Restore.
🔴 Search not finding emails
In “This Label” mode, search only covers emails loaded in the current view. Switch to “All Emails” mode (click the scope button) and press Enter to search across your entire Gmail account. Global search is capped at 100 results.
🔴 Pagination showing wrong count
Stats always show totals for ALL emails, regardless of pagination. The page count and "Page X of Y" indicator reflect the current filtered or full set. This is by design.

⚠️ 18. Known Limitations

The following are known limitations of the current system. These are accepted trade-offs, not bugs.

🟡 API keys hardcoded in public repo
The Google Client ID, API Key, and Airtable tokens are embedded in the public GitHub repository HTML file. This is a known security trade-off for simplicity.
🟡 Basic AI learning system
AI learning is based on a lookup table of corrections, not sophisticated machine learning. It stores past corrections and feeds them into future prompts.
🟡 Gmail API rate limiting workaround
Rate limits are handled with 300ms delays between requests. This is a simple workaround, not proper exponential backoff with retry logic.
🟡 Single HTML file architecture (~150KB)
The entire app is a single HTML file with embedded CSS and JavaScript. This works but becomes harder to maintain as it grows.
🟡 No collaborative features
The app is designed for single-user operation. While data syncs across devices via Cloudflare KV, there is no multi-user support or collaboration.
✅ Global search available
Local search (“This Label”) only covers emails loaded in the current view. Use Global search (“All Emails”) to search across your entire Gmail account. Global search is capped at 100 results and may return emails outside the 9 numbered labels.
🟡 Single-level undo only
Only the most recent label move can be undone. There is no multi-step undo history.
🟡 Audit log limited to 500 entries
The audit log stores a maximum of 500 entries in localStorage. Older entries are trimmed automatically. Export as CSV before clearing if you need a full history.
